# SpeakoFlow — Release Plan (detailed)

The goal of this doc: get SpeakoFlow from "3-week fork with rough edges" to a
**shippable v1 open-source release**, by splitting the work into tracks that
can run across **multiple parallel AI sessions** without stepping on each other.

Read this with [`RELEASE_SIMPLE.md`](./RELEASE_SIMPLE.md) — that one is the
plain-English "just tell me what to do" version. This one is the reference.

> Companion docs: architecture in [`../AGENTS.md`](../AGENTS.md), current state
> in [`PROGRESS.md`](./PROGRESS.md), prompt map in [`prompts-reference.md`](./prompts-reference.md).

---

## 0. Architecture snapshot (so any session has context)

**Stack:** Tauri 2.x. Rust backend (`src-tauri/src`), React + TypeScript frontend
(`src`). **Three separate windows / Vite entry points:**

| Window | Frontend entry | Purpose |
|--------|---------------|---------|
| Main settings | `src/App.tsx` + `src/components/**` | all configuration UI |
| Recording overlay | `src/overlay/**` | small "recording…" indicator |
| Assistant panel | `src/assistant/**` | floating glass chat panel |

**Backend (managers own the state):** `managers/{audio,model,transcription,history}.rs`,
plus `assistant.rs`, `tts.rs`, `web_search.rs`, `screenshot.rs`, `secret_store.rs`
(OS keychain), `settings.rs` (central, 1745 lines), `actions.rs` (dictation pipeline),
`tray.rs`, `overlay.rs`, `shortcut/`, `lib.rs` (setup + tray menu event handler ~L225 +
command registration).

**Data flow:** hotkey → record → VAD → local STT (Whisper/Parakeet/Moonshine) →
either paste (dictation) or LLM → panel (assistant). Settings live in Zustand
(`stores/settingsStore.ts`) → Tauri command → Rust `settings.rs` → `tauri-plugin-store`.
Secrets live in the OS keychain via `secret_store.rs`.

### Shared "hotspot" files — the merge-conflict map

These files are touched by many tasks. **Coordinate edits to these or you will
get conflicts across parallel sessions:**

| File | Why it's hot | Rule for parallel work |
|------|--------------|------------------------|
| `src/i18n/locales/en/translation.json` | every UI task adds strings | append-only; conflicts are trivial to resolve |
| `src/bindings.ts` | auto-generated by tauri-specta | **never hand-merge** — regenerate after backend changes land |
| `src-tauri/src/settings.rs` | many tasks add setting fields | add fields with `#[serde(default)]` + a `default_*` fn; append, don't reorder |
| `src-tauri/src/lib.rs` | command registration + tray event handler | keep edits to distinct line regions; expect small merges |
| `src-tauri/src/assistant.rs` | panel, screenshot, providers all touch it | **serialize** tasks that edit it (see Track PANEL vs ASSISTANT-CFG) |

## 1. How to run parallel sessions safely

1. **One track = one branch = one session.** Use a branch per track
   (`brand/…`, `backend/…`, `panel/…`). Optionally `git worktree` so several
   checkouts exist at once.
2. **Never run two sessions that edit the same hotspot file at the same time.**
   Use the conflict matrix in §3.
3. **Backend command changes regenerate `bindings.ts`.** Whoever lands a backend
   command last regenerates bindings; frontend tracks rebase onto it.
4. **Every task ends green:** `bun run lint` + `bun run format:check`, and
   `bun run tauri dev` compiles. New settings get `#[serde(default)]`. New UI
   strings go through i18n (ESLint enforces it).
5. **Give each session the task block verbatim** from §4 as its prompt (goal,
   files, depends-on, done-when). That is what makes them independent.

## 2. Release scope decision (do this first, 30 min, human)

Before any code, lock these down — several tasks depend on them:

- **Final app name:** SpeakoFlow (already in `productName` + `Wordmark`). ✅ keep.
- **Bundle identifier:** currently `com.pais.handy` — **must change** (e.g.
  `com.abhishekbarali.speakoflow`). This is the keychain service name in
  `secret_store.rs` and the app-data folder, so pick the final value **once**;
  changing it later orphans users' saved keys and settings.
- **First public version:** currently `0.8.3`. Decide `0.1.0` (honest "early")
  or `1.0.0` (confident). Set in `tauri.conf.json` + `Cargo.toml` + `package.json`.
- **Code signing:** you cannot use cjpais's cert (it's in `signCommand`). Decide:
  ship unsigned (document SmartScreen/Gatekeeper warnings), or self-sign, or
  buy a cert later. This gates the installer track.
- **Scope line:** v1 = the three base jobs done well (dictation, assistant,
  screen-vision) + clean branding + trustworthy install/update. Everything
  labelled *(post-v1)* below is explicitly out of the first release.

## 3. The six tracks and their conflict matrix

Each track "owns" a set of files. Tracks that own disjoint files run fully in
parallel.

| Track | Owns (primary files) | Type |
|-------|----------------------|------|
| **BRAND** | `tauri.conf.json`, `Cargo.toml`, `package.json`, `src-tauri/icons/**`, `src-tauri/resources/**`, `src-tauri/nsis/**`, `tray.rs`, `tray_i18n.rs`, `about/AboutSettings.tsx`, `update-checker/**`, `README/CONTRIBUTING/BUILD/AGENTS`, `.github/**` | config + docs + assets |
| **BACKEND** | `managers/history.rs`, `managers/model.rs`, `web_search.rs`, `tts.rs`, `llm_client.rs` (+ verification of `assistant.rs`) | Rust correctness |
| **SETTINGS-IA** | `Sidebar.tsx`, `App.tsx`, `settings/general/**`, `settings/advanced/**`, `settings/history/**`, section wrappers | frontend IA |
| **PANEL** | `src/assistant/**`, `screenshot.rs`, `overlay.rs` (panel sizing) | assistant window |
| **ASSISTANT-CFG** | `settings/assistant/AssistantSettings.tsx`, prompt/persona logic | assistant config UI |
| **DESIGN** | logo/icon source assets, `App.css` / global tokens, visual polish | design system |

**Conflict matrix** (✅ safe in parallel, ⚠️ coordinate, ❌ do not overlap):

|             | BRAND | BACKEND | SETTINGS-IA | PANEL | ASSISTANT-CFG | DESIGN |
|-------------|:-----:|:-------:|:-----------:|:-----:|:-------------:|:------:|
| BRAND       | —     | ⚠️ lib.rs | ✅ | ✅ | ✅ | ⚠️ icons |
| BACKEND     |       | —       | ✅ | ⚠️ assistant.rs | ⚠️ assistant.rs | ✅ |
| SETTINGS-IA |       |         | —           | ✅ | ⚠️ i18n | ⚠️ CSS |
| PANEL       |       |         |             | —     | ⚠️ assistant.rs | ⚠️ CSS |
| ASSISTANT-CFG |     |         |             |       | —             | ⚠️ CSS |
| DESIGN      |       |         |             |       |               | —      |

Takeaway: **BRAND + BACKEND + PANEL + DESIGN can all run at once.** SETTINGS-IA
and ASSISTANT-CFG both touch frontend + i18n, so pair them carefully. `assistant.rs`
is the one Rust file three tracks want — serialize it.

## 4. Tasks by phase (each block = one AI session)

Effort: **S** ≈ <2h, **M** ≈ half-day, **L** ≈ 1–2 days. `∥` = parallel-safe.

### Phase A — Foundation & branding (start immediately)

#### A1. Bundle identifier + config rebrand — Track BRAND — **M**

- **Files:** `tauri.conf.json` (`identifier`, `signCommand`, updater), `Cargo.toml`,
  `package.json`, `src-tauri/src/secret_store.rs` (service name), any `handy.log`
  log-file name, `HANDY_NO_GTK_LAYER_SHELL` env var, VAD download URL
  `blob.handy.computer` (verify it still resolves or self-host).
- **Depends on:** §2 decisions (identifier, version, signing).
- **Done when:** fresh `bun run tauri dev` launches under the new identifier;
  keychain writes land under the new service name; no `com.pais.handy` left in
  `src-tauri/**` outside comments.
- **Watch:** changing the identifier resets keychain + settings for existing dev
  installs — expected. Re-enter API keys after.

#### A2. Logo + icon set — Track DESIGN — **M** — `∥`
- **Goal:** replace the Handy hand/waveform mark with a SpeakoFlow mark.
- **Files:** produce a master SVG/1024px PNG, then regenerate the full

- **Done when:** all icon sizes regenerated; tray shows the new mark in

- **Watch:** tray icon paths are hard-coded in `tray.rs::get_icon_path` — if you
  rename files, update that map (coordinate with A1/A4).

#### A3. Docs + repo hygiene rebrand — Track BRAND — **S** — `∥`
- **Goal:** README, CONTRIBUTING, BUILD, AGENTS, `.github/**` reflect SpeakoFlow,

  required attribution; README has a real feature list + screenshots.

#### A4. Tray "Home" + version label + About cleanup — Track BRAND — **S**
- **Goal:** add a **Home** item to the tray that opens the main window; make the

- **Files:** `tray.rs` (add a `home` MenuItem alongside `settings`),

- **Depends on:** A1 (naming) ideally, but can proceed independently.
- **Done when:** tray shows **Home** and it focuses the settings window; About

### Phase B — Base-feature correctness (parallel with Phase A)

#### B1. Fix history limit + retention — Track BACKEND — **M** — `∥`
- **Goal:** make "Advanced → History limit" and "Recording retention" actually
  prune. The wiring exists (`settings.rs` fields, `commands/history.rs`,
  enforcement ~`managers/history.rs:368`) — this is a **bug fix**, not new build.
- **Steps:** trace `update_history_limit` → does the manager prune on write and
  on startup? Verify count-based vs time-based retention both fire. Add a test.
- **Done when:** setting limit=5 with 10 entries prunes to 5; retention period
  deletes old WAVs; verified by test + manual.

#### B2. Provider audit matrix — Track BACKEND — **L** — `∥` (splits into sub-sessions)
- **Goal:** confirm each provider path is correct even without keys, using
  Firecrawl to check current API docs (auth header, endpoint shape, model param).
- **Providers to verify:**
  - **Assistant LLM** (`assistant.rs`, `llm_client.rs`): OpenAI, Azure OpenAI,
    Groq, OpenRouter, Anthropic, local (Ollama/LM Studio), custom.
  - **Web search** (`web_search.rs`): Serper, Brave, Tavily, Exa, SerpAPI.
  - **TTS** (`tts.rs`): Kokoro (local), OpenAI-compatible, ElevenLabs.
- **Method:** for each, Firecrawl the official API reference; diff the request
  our code builds (URL, auth, body, response parse) against current docs; note
  drift; fix cheap ones; file issues for keyed ones you can't test live.
- **Output:** a `docs/PROVIDER_AUDIT.md` matrix (provider · endpoint · auth ·

- **Done when:** matrix complete; keyless providers (DuckDuckGo/Serper free tier,

## 5. Parallelization waves (the actual schedule)

Run each wave's sessions **at the same time**; finish + merge a wave before the
next where noted.

**Wave 1 — kick off now (4 sessions, disjoint files):**
- S1 = A1 (identifier/config)  ·  S2 = A2 (logo/icons)  ·  S3 = B1 (history fix)  ·  S4 = B2 (provider audit)

**Wave 2 — after A1 merges (identifier stable):**
- S1 = A3 (docs) ∥ A4 (tray Home)  ·  S2 = B3 (model registry)  ·  S3 = D1 (panel size)  ·  S4 = D2 (screenshot, serialize assistant.rs after B2)

**Wave 3 — feature/IA build:**
- S1 = C1 (settings IA)  ·  S2 = C2 (model UI, needs B3)  ·  S3 = D3a (split AssistantSettings)  ·  S4 = B4 (optimization)

**Wave 4 — polish + assistant config:**
- S1 = E1 (visual polish, needs C1)  ·  S2 = D3b (personas) + D4 (preview)  ·  S3 = E2 (onboarding)

**Wave 5 — release infra + QA (see §6, mostly sequential):**
- installer/signing → updater/latest.json → full QA sweep → release cut.

Rule of thumb: **4 concurrent sessions** is the sweet spot (one per disjoint
track). Going wider mostly adds merge overhead on the hotspot files in §0.

## 6. Pre-release QA & security checklist (the "what else to test")

Things beyond your list that **must** be checked before publishing.

### Security (Track BRAND/BACKEND)
- [ ] **CSP is `null`** in `tauri.conf.json` → set a real Content-Security-Policy.
- [ ] **`assetProtocol.scope` is `["**"]`** (any file readable) → tighten to the
      recordings/models dirs only.
- [ ] Secrets: confirm keychain storage works on Win/mac/Linux and **no key is
      ever written to `settings_store.json` or logs**.
- [ ] `[run]` magic command in text replacements is **off by default** and warns.
- [ ] Network egress review: list every outbound call (HF model downloads,
      provider APIs, web-search APIs, VAD download, GitHub update check) and put
      it in a PRIVACY section of the README. No silent telemetry.
- [ ] `cargo audit` + `bun audit` clean (or documented).
- [ ] Updater private key is stored securely and **never committed**; pubkey in
      `tauri.conf.json` matches it.

### Functional QA
- [ ] Fresh install → onboarding → first dictation works end to end.
- [ ] Dictation into 3+ real apps (browser, editor, chat) pastes correctly.
- [ ] Assistant: voice ask, typed ask, screen-vision, TTS, web search — each on
      at least one provider.
- [ ] All global shortcuts rebind + persist; document conflicts (Ctrl+Space vs
      IME/Spotlight).
- [ ] Empty/error states: no model, no mic, no network, bad API key, provider
      down — all show a friendly message, never a crash.
- [ ] Model download → cancel → resume → delete → switch active model.
- [ ] History: add, prune by limit, prune by retention, save/star, delete, retry.
- [ ] Light + dark theme across every screen; RTL doesn't break layout.
- [ ] i18n: English complete; other locales fall back gracefully.

### Cross-platform build (blocking for the platforms you ship)
- [ ] Windows build + NSIS installer installs, runs, updates, uninstalls clean.
- [ ] macOS build (if shipping): Gatekeeper behavior documented (notarize or
      workaround). Accessibility + mic permission prompts fire.
- [ ] Linux build (if shipping): deb/AppImage runs; GTK layer-shell overlay works
      (and the `*_NO_GTK_LAYER_SHELL=1` fallback).

### Release cut (sequential, last)
1. Version bump in `tauri.conf.json` + `Cargo.toml` + `package.json`.
2. Update `PROGRESS.md` + changelog.
3. Tag + GitHub release; CI builds signed artifacts + `latest.json`.
4. Verify auto-update: install old build → check for updates → updates to new.
5. README screenshots current; LICENSE + attribution present.
