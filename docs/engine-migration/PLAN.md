> **This is the single source of truth for the transcription-engine migration.**
> The executor AI MUST read this file at the start of every session, do exactly one
> session, verify it against the Acceptance Criteria, then update this file (tick the

> See `README.md` for the paste-prompt → implement → verify → tick loop, and `PROMPTS.md`

> **Granularity note:** this is deliberately a small number of *coherent, end-to-end* sessions,
> not many micro-tasks — a capable agent does better holding related work in one context. Each

## 1. Goal & non-negotiables

**Goal:** adopt Handy's native `transcribe.cpp` (ggml/GGUF) engine, ship the new GGUF models (streaming + batch) as the **recommended** set, get **real native streaming** (replacing the VAD-chunked hack), add an **optional user-toggleable live-transcription window**, and structure the transcription core so future Handy updates are a cheap pull.

**Non-negotiables (hard constraints):**
- **N1 — Never break the working app.** At every session boundary it must still build and dictate. The existing `transcribe-rs 0.3.11` engine stays functional until the new one is proven.
- **N2 — Side-by-side, additive.** Add `transcribe-cpp` as a *new* engine next to `transcribe-rs`; don't rip out the old path. New GGUF models use the new engine; existing models keep the old one.
- **N3 — Everything gated / default-off** until proven. New settings default `false`.
- **N4 — Transcription-core only.** Do not touch `assistant.rs`, `memory.rs`, `tts.rs`, `web_search.rs`, `screenshot.rs`, `llm_client.rs`, history storage. (Verified independent — §3.)
- **N5 — No commits unless the human asks.** No force-push, no dependency rewrites beyond a session's scope.

## 2. Before → After architecture

**Before:** `transcribe-rs 0.3.11` (`["whisper-cpp","onnx"]`; Win +`whisper-vulkan`,`ort-directml`; mac `whisper-metal`; Linux `whisper-vulkan`). `LoadedEngine` (8 variants); hardcoded catalog in `model.rs`; homegrown `StreamRouter`+`VadChunked`+`EnergyVad` streaming (isolated-chunk quality ceiling, `tentative` always empty).

**After:** `transcribe-rs 0.3.11` **unchanged** + `transcribe-cpp 0.1.2` added for GGUF models. New `EngineType::TranscribeCpp` + `LoadedEngine::TranscribeCpp(Session)`; GGUF single-file models from `huggingface.co/handy-computer/<slug>-gguf`. Real native streaming (`Session::stream`→`feed`→`finalize`, genuine committed+tentative), gated to models whose GGUF advertises `supports_streaming`. Optional live window (Handy's resized 400×120 streaming card). Recommended set mirrors Handy (Parakeet Unified EN 0.6B #1, Nemotron Streaming 3.5 #2, …).

**Whisper-stack end-state:** **Shape (b) is the default** (keep `transcribe-rs 0.3.11` as-is, add `transcribe-cpp` only for new GGUF models — non-breaking, ships two whisper stacks). **Shape (a)** (mirror Handy exactly: `transcribe-rs` → `0.3.8` ONNX-only, whisper via `transcribe-cpp`, drop `ort-directml`) is an **optional convergence inside Session 6**, deferred until the new engine is proven.

## 4. Key upstream facts (reference while implementing)

- **Crate:** `transcribe-cpp = "0.1.2"` (safe binding; FFI in `transcribe-cpp-sys`). Features `default=["metal"]`, others `metal|vulkan|cuda|openmp|shared|dynamic-backends`. **Use `default-features=false` off macOS.**
- **Batch:** `Model::load_with(&path,&ModelOptions{backend,gpu_device})?` → `.session()?` → `session.run(&pcm,&RunOptions)? -> Transcript{text,..}`. PCM = **mono f32 @16 kHz, [-1,1]** (no built-in resampler).
- **Streaming:** `let mut stream = session.stream(&run_opts,&StreamOptions::default())?;` → `let upd = stream.feed(&pcm)?;` (if `upd.committed_changed||upd.tentative_changed` → `stream.text()` = `{committed, tentative}`) → `stream.finalize()?` then `stream.text().display()`; cancel = `stream.reset()`. `CommitPolicy::{Auto,OnFinalize,StablePrefix}` (+`stable_prefix_agreement_n`, default 3).
- **Options:** `RunOptions{ task:Task::{Transcribe,Translate}, language:Option<String>, target_language:Option<String>, timestamps:TimestampKind, pnc, itn, family:Option<RunExtension> }`. Custom words/initial prompt = `RunExtension::Whisper(WhisperRunOptions{initial_prompt})` **whisper-arch only** (non-whisper rejects it → fuzzy post-correction, same as today).
- **Capabilities:** `session.model().capabilities() -> Capabilities{ supports_streaming, supports_translate, languages, native_sample_rate, ... }`. Decide batch-vs-stream from `supports_streaming` **at load**, reconciled with the GGUF header.
- **Backends:** `init_backends_default()` at startup; `devices()->Vec<Device>`; `Backend::{Auto,Cpu,Metal,Vulkan,Cuda}`.
- **Catalog schema** (`catalog.json`, `catalog_version:1`): per model `id`(`handy-computer/<slug>-gguf`), `slug,name,architecture,family,parameters,description,base_model,license,language_count,languages[],capabilities{streaming,translate,lang_detect,timestamps},speed_score,accuracy_score,files[{filename,quant,size_bytes}],default_quant,recommended,recommended_rank`. Quants `F32/F16/BF16/Q8_0/Q6_K/Q5_K_M/Q4_K_M`. (Re-fetch live from `raw.githubusercontent.com/cjpais/Handy/main/src-tauri/src/catalog/catalog.json` in Session 3.)
- **GGUF probing:** `gguf_meta.rs` (dep-free GGUF v2/v3 LE parser, magic `0x46554747`) + `model_capabilities.rs` read `general.architecture`, `stt.variant`, `general.languages`, `stt.capability.{streaming,translate,lang_detect}`. Parakeet streaming inferred by the native loader when KV absent ("never guess" → reconcile post-load).
- **Live window:** Handy reuses the ONE overlay window, **resized** per state — streaming `400×120` (`OVERLAY_STREAM_WIDTH/HEIGHT`), compact `256×46`. Typed `StreamTextEvent{committed,tentative}` + `StreamPhaseEvent{phase:Listening|Working,kind:Transcribing|Polishing}`.
- **Windows build:** features `["dynamic-backends","vulkan"]`; needs **Vulkan SDK 1.4.309.0**, **SPIRV-Headers** (`vcpkg install spirv-headers:x64-windows` + `CMAKE_PREFIX_PATH`), **cmake**, and **`CARGO_TARGET_DIR` path-shortening** (`<drive>\t`, CMake paths exceed MAX_PATH). Ships loadable backend DLLs → sign each. Upstream **dropped `ort-directml`** (pyke AVX2 prebuilt crashed pre-Haswell CPUs); links baseline ONNX Runtime 1.24.2 dynamically.
- **Recommended set (mirror Handy ranks):** ①`parakeet-unified-en-0.6b` (EN, **streaming**, Q8_0≈697MB, acc90) ②`nemotron-3.5-asr-streaming-0.6b` (28 langs, **streaming**, Q8_0≈716MB, acc82) ③`canary-180m-flash` (en/de/es/fr, batch, Q8_0≈208MB, acc88) ④`cohere-transcribe-03-2026` (14 langs, batch, Q5_K_M≈1.65GB, acc92) ⑤`whisper-medium` (99 langs, batch, Q8_0≈793MB, acc84). All streaming-capable: those 2 + `nemotron-speech-streaming-en-0.6b`, `moonshine-streaming-{tiny,small,medium}`, `voxtral-mini-4b-realtime` (heavy 3GB).

## 5. Session flow & parallelization

```
Session 1  Spike & isolation proof         (GO/NO-GO — do first, standalone)
   ↓
Session 2  Batch engine integration        (types + load + transcribe + backend init)
   ↓
Session 3  Catalog + capability + model UI  ├─ 3a backend (catalog+probe) ─┐ optional
   ↓                                         └─ 3b frontend (model UI) ─────┘ parallel split
Session 4  Real native streaming            (worker + tentative render + retire hack)
   ↓
Session 5  Optional live-transcription window (setting + resized overlay card)   [needs S4 events]
Session 6  Recommended defaults + legacy    (+ optional upstream convergence)     [needs S3]
Session 7  Build / installer / CI / signing + FOLLOW_HANDY.md   [prep alongside S3–S5, finalize last]
```

- The core (1→2→3→4) is **intentionally sequential** — each builds directly on the last, and one agent carries the context forward (that's the point).
- **Parallel opportunities:** Session 3 can split into **3a backend ∥ 3b frontend UI**; Session 7's build config can be **drafted alongside Sessions 3–5** and validated last. Session 5's setting-plumbing is independent but its window needs Session 4's `stream-text` events.
- Within any session, the agent does all **Sub-steps** together in one context (do NOT split them into separate runs unless a sub-step is explicitly marked parallel).
